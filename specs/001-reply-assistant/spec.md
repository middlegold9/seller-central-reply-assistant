# Spec: Seller Central 买家消息 AI 回复助手

> Spec-Kit 规格说明（做什么 / 为什么）。技术细节见 `plan.md` / `tasks.md`。

## 目标
在 Seller Central 买家消息页直接生成「合规、贴合品牌、带政策风险提示」的回复草稿，人工确认后发送。最「当场击穿」的 Demo：客服时差 + 量大的刚需，打开后台就有 AI 草稿。

## 用户故事
- **US1 (P0)** — 作为跨境客服，我希望在消息页点「AI 草稿」就拿到可直接发的回复，不用切到别的工具。
- **US2 (P0)** — 作为客服，我希望草稿**自动避开政策红线**（不索评、不引导站外），违规意图被标红或拦截。
- **US3 (P1)** — 作为卖家，我希望用买家语言（英/西/德/日…）回复，提升体验。
- **US4 (P1)** — 作为客服，我希望系统对法律/威胁/A-to-z/差评移除请求**自动转人工**并给摘要。
- **US5 (P2)** — 作为代运营，我希望多店铺切换不同品牌语气/模板。
- **US6 (P2)** — 作为运营，我希望从历史高采用率回复中学习，提升话术质量。

## 功能需求
- **FR1** MUST 在 Buyer Messages 页注入「AI 草稿」入口（Content Script）。
- **FR2** MUST 解析当前会话（买家消息、订单号、ASIN、语言）。
- **FR3** MUST 生成结构化草稿 `{draft, language, suggested_actions[], risk_flags[], confidence}`。
- **FR4** MUST 护栏引擎对「买家消息」和「AI 草稿」各跑一遍，致命违规阻断发送、警告标红。
- **FR5** MUST 高风险类别（法律/威胁/A-to-z/差评移除）转人工，不生成发送草稿。
- **FR6** SHOULD 多语言生成（检测买家语言 → 同语言回复）。
- **FR7** SHOULD 可调用 `sp-api-mcp-server` 丰富上下文（订单状态/物流/买家信息）。
- **FR8** MUST 仅把草稿填回官方输入框，不模拟提交。
- **FR9** MUST 设置页可配 API Key（本地）、品牌名/语气、退款上限、护栏严格度。
- **FR10** SHOULD 发送后脱敏记录到学习库（RAG）。

## 验收清单
- [ ] 消息页出现「AI 草稿」按钮并能拿到草稿。
- [ ] 构造违规意图（"给折扣换好评"），护栏阻断/标红（合规率 ≥99%）。
- [ ] 高风险会话转人工卡片出现。
- [ ] 草稿填入官方输入框，未绕过发送。
- [ ] 设置页 API Key 存本地，不出现在网络明文。

## 范围外
- 自动发送、自动退款执行；替代亚马逊消息系统；v1 多店铺/Web Store 上架。
